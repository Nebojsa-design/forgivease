Overview

ForgivEase helps users name what hurts, complete guided forgiveness practices, record optional local voice notes, set reminders, review simple insights, and delete local data. The current MVP keeps practice history on the user's device.

Data Stored On The Device

The current MVP can store these items locally:

  • Practice title, selected path, focus type, completion date, and reflection note.
  • Before and after emotion ratings for completed practices.
  • Optional voice-note metadata and local audio path when a user records a voice note.
  • Reminder settings, reminder message choice, daily time, and test interval setting.
  • Onboarding completion state and local draft recovery state.
  • Local export text prepared by the user.

Data Transmission

The current MVP has no account system, backend sync, analytics SDK, ads, tracking SDK, paid products, live AI calls, or cloud media upload. Practice history, ratings, notes, voice notes, and reminder settings stay on the device in the current MVP runtime.

Permissions

ForgivEase may request these permissions:

  • Microphone permission, used when the user chooses to record a private voice note.
  • Notification permission, used when the user chooses reminders.

Users can deny these permissions. The core text practice remains available.

Local Data Deletion

Users can delete local practice history in Settings through Delete local data. Users can also remove a voice note from an individual saved process. Native builds attempt to remove the local audio file when a voice note, saved process, or all local data is deleted.

Sensitive Content

ForgivEase may contain private user-entered reflections about emotions, conflict, guilt, regret, fear, boundaries, or personal situations. The current MVP keeps this content local to the device.

Users should use qualified human support for urgent danger, violence, abuse, self-harm risk, medical care, legal decisions, or severe distress.

Children

ForgivEase is intended for teens and adults who can use private reflection tools responsibly. Store age rating answers should reflect the final app content, wellness framing, spiritual content, voice recording, and any future community features.

Service Providers

Development builds use Expo and EAS tooling for app builds, internal distribution, and development-client workflows. App stores may process installation, purchase, crash, or account information according to Apple and Google terms.

Future Updates

This policy should be updated before ForgivEase adds accounts, backend sync, analytics, AI features, payments, group rooms, moderation tools, cloud storage, or public launch infrastructure.
